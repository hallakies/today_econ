# 오늘경제 발행 품질 체크리스트

이 문서는 자동 발행 성공보다 독자가 이해할 수 있는 정확한 편집물을 우선하기 위한
회귀 방지 기준이다. 모든 게시물은 아래 항목을 통과해야 한다.

## 1. 입력 무결성

- [ ] 기사 본문에서 기자명, 입력 시각, 검색 유도, 공유 버튼, 관련 기사 문구를 제거한다.
- [ ] `기자`, `입력`, `Google 검색`, `선호 추가`, `알아보기`가 카드에 노출되지 않는다.
- [ ] 제목이 본문 첫 문장으로 중복돼도 핵심 사실로 다시 사용하지 않는다.
- [ ] 기사 본문에서 주제와 연결된 검증 가능한 사실을 최소 2개 확보한다.

## 2. 하나의 이야기

- [ ] 생성 전에 핵심 주제, 영향을 받는 독자, 돈의 경로를 한 번만 결정한다.
- [ ] 모든 카드가 같은 핵심 주제와 돈의 경로를 공유한다.
- [ ] 주택·청약 기사에 근거 없는 자영업·노후·대출 일반론을 끼워 넣지 않는다.
- [ ] 기사에 없는 숫자, 날짜, 정책, 인과관계를 만들지 않는다.
- [ ] 불확실한 영향은 가능성으로 표현하고 결과를 바꾸는 조건을 밝힌다.

## 3. 세 장 편집

- [ ] 기본 구성은 표지, 사건과 이유, 내 돈 영향과 한 가지 행동의 3장이다.
- [ ] 표지는 원문 제목을 임의 길이로 자르지 않고 완결된 주장으로 다시 쓴다.
- [ ] 표지에 말줄임표를 사용하지 않는다.
- [ ] 필요한 개행은 의미 단위로 넣고, 조사나 수치 중간에서 자르지 않는다.
- [ ] 두 번째 장에는 기사에서 확인된 사실만 2개 넣는다.
- [ ] 세 번째 장에는 독자 영향 2개와 구체적인 확인 행동 1개만 넣는다.
- [ ] 별도의 네 번째 장은 독립적인 정보 가치가 검증된 경우에만 허용한다.

## 4. 의미 품질 게이트

- [ ] 웹페이지 메타데이터와 검색 UI 문구를 발견하면 게시를 중단한다.
- [ ] 표지가 완결되지 않았거나 말줄임표로 끝나면 게시를 중단한다.
- [ ] 카드 사이 핵심 주제가 바뀌면 게시를 중단한다.
- [ ] 형식 통과가 의미 정확성을 대신하지 않는다.
- [ ] 선택한 기사로 일관된 원고를 만들 수 없으면 품질을 낮추지 않고 다음 기사 후보로 넘어간다.

## 5. 릴스와 배포

- [ ] 카드별 글자 수에 따라 6~12초의 읽기 시간을 계산한다.
- [ ] 외부 음원이 짧아도 영상이 중간에 잘리지 않는다.
- [ ] Instagram 캡션에는 클릭할 수 없는 원문 URL을 넣지 않는다.
- [ ] 원문 URL은 Slack 운영 정보와 게시 이력에만 보존한다.
- [ ] 해시태그는 한 곳에서 한 번만 붙인다.
- [ ] 점이 들어간 `#today.econ` 대신 `#today_econ`을 사용한다.
- [ ] 같은 해시태그가 중복되면 게시 전에 제거한다.

## 6. 독자 경험 대원칙

- [ ] 이 계정 하나를 보면 중요한 경제 소식을 놓치지 않는다는 느낌을 준다.
- [ ] 경제 초보도 한 번에 읽고 이해할 수 있는 문장과 화면을 제공한다.
